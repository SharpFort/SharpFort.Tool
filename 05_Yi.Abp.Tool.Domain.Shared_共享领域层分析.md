# Yi.Abp.Tool.Domain.Shared - 共享领域层分析

## 📌 概述

**Yi.Abp.Tool.Domain.Shared** 是 ABP 分层架构中的**共享领域层**（Domain Shared Layer），存放被各层共享的类型定义，包括枚举、DTO、配置选项等。遵循 DDD 原则——不包含业务逻辑，只定义数据结构。

**项目路径:** `tool/Yi.Abp.Tool.Domain.Shared/`

## 🏗️ 项目信息

| 属性 | 值 |
|------|-----|
| **TargetFramework** | net8.0 |
| **输出类型** | Class Library（类库） |

## 📦 项目依赖

| 依赖 | 说明 |
|------|------|
| `Yi.Framework.Core` | Yi 框架核心模块 |
| `Yi.Framework.Mapster` | Yi 框架 Mapster 映射集成 |

### 2. TemplateGenCreateDto.cs - 模板生成领域 DTO

**文件路径:** `Yi.Abp.Tool.Domain.Shared/Dtos/TemplateGenCreateDto.cs`

```csharp
public class TemplateGenCreateDto
{
    public string Name { get; set; }                      // 模块名称
    public string GiteeRef { get; set; }                  // Gitee 分支引用
    public DbmsEnum Dbms { get; set; }                    // 数据库类型
    public Dictionary<string, string> ReplaceStrData { get; set; }  // 替换字典

    public void SetTemplateGiteeRef(string moduleType)
    {
        this.GiteeRef = moduleType.ToLower();  // 分支名转小写
    }
}
```

**与 Contracts 层 DTO 的区别:**

| 属性 | TemplateGenCreateInputDto | TemplateGenCreateDto |
|------|:---:|:---:|
| Name | ✅ | ✅ |
| ModuleSoure | ✅ | → 转为 GiteeRef |
| Dbms | ✅ | ✅ |
| ReplaceStrData | ✅ | ✅ |
| GiteeRef | ❌ | ✅ (由 ModuleSoure 转换而来) |

**数据转换路径:**
```
ModuleSoure (用户输入)
    ↓ ToLower()
GiteeRef (Gitee 分支引用)
    ↓ 用于 GiteeManager API 调用
```

### 3. DbmsEnum.cs - 数据库类型枚举

**文件路径:** `Yi.Abp.Tool.Domain.Shared/Enums/DbmsEnum.cs`

```csharp
public enum DbmsEnum
{
    MySQL,        // MySQL 数据库
    SQLite,       // SQLite 数据库
    SQLServer,    // SQL Server 数据库
    Oracle,       // Oracle 数据库
    PostgreSQL    // PostgreSQL 数据库
}
```

**支持 5 种主流数据库:**

| 枚举值 | 数据库 | 说明 |
|--------|--------|------|
| `MySQL` | MySQL | 开源关系型数据库 |
| `SQLite` | SQLite | 嵌入式数据库 |
| `SQLServer` | SQL Server | 微软关系型数据库 |
| `Oracle` | Oracle | 甲骨文数据库 |
| `PostgreSQL` | PostgreSQL | 开源关系型数据库 |

**使用场景:**
```bash
# CLI 命令中通过 -dbms 选项指定
yi-abp new MyModule -dbms MySQL
yi-abp new MyModule -dbms PostgreSQL
```

> ⚠️ **注意:** 当前代码中，`dbms` 选项仅输出提示信息，实际模板替换逻辑尚未完全集成数据库枚举。用户需在生成后手动修改 `DbConnOptions:Url` 及 `DbType`。

### 4. ToolOptions.cs - 工具配置选项

**文件路径:** `Yi.Abp.Tool.Domain.Shared/Options/ToolOptions.cs`

```csharp
public class ToolOptions
{
    /// <summary>
    /// 临时文件目录
    /// </summary>
    public string TempDirPath { get; set; }
}
```

**配置文件映射（appsettings.json）:**
```json
{
  "ToolOptions": {
    "TempDirPath": "wwwroot/temp",
    "ModuleTemplateFilePath": "wwwroot/ModuleTemplate.zip",
    "ProjectTemplateFilePath": "wwwroot/ProjectTemplate.zip"
  }
}
```

**当前已使用配置项:**

| 配置项 | 类型 | 说明 |
|--------|------|------|
| `TempDirPath` | `string` | 临时文件存放目录 |

**配置项在 appsettings.json 中存在但代码中未显式绑定:**
| 配置项 | 说明 |
|--------|------|
| `ModuleTemplateFilePath` | 模块模板文件路径（预留） |
| `ProjectTemplateFilePath` | 项目模板文件路径（预留） |

> 💡 **改进建议:** 可以考虑在 `ToolOptions` 类中添加 `ModuleTemplateFilePath` 和 `ProjectTemplateFilePath` 属性，以便通过配置直接指定本地模板路径。
