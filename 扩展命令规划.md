# SharpFort.Tool 扩展命令规划

> 状态: 方案讨论完成，待实施

---

## 已确认不做的功能

### config 命令 --- 不做

原因: 99% 用户直接用 SharpFort/SharpFort.Template，零配置。高级用户手动编辑 ~/.sharpfort/config.json 即可。

---

## 计划新增

### 1. list 子命令扩展 --- 缓存管理

| 命令 | 说明 |
|------|------|
| `sharpfort new list --refresh` | 强制刷新所有缓存 |
| `sharpfort new list --clear` | 清空所有缓存 |

设计原则: 不新增子命令，在现有 list 下扩展 flag。

---

### 2. init 命令 --- 首次引导

引导用户克隆 SharpFort.Net 框架项目，框架包含 RBAC/审计日志/限流/多租户/定时任务。

交互流程:
  1. 提示 git clone https://github.com/SharpFort/SharpFort.Net
  2. 说明框架提供的基础能力
  3. 模板仓库 Owner 确认（默认 SharpFort）
  4. GitHub Token（可选）
  5. 自动生成 ~/.sharpfort/config.json


### 3. doctor 命令 --- 环境诊断 + 双源自动切换

背景: 国内用户可能无法访问 GitHub API，实现主备双源自动切换。

配置结构 (config.json):
```json
{
  "Repo": {
    "Primary": {
      "Host": "https://api.github.com",
      "Owner": "SharpFort",
      "RepoName": "SharpFort.Template"
    },
    "Fallback": {
      "Host": "https://gitee.com/api/v5",
      "Owner": "SharpFort",
      "RepoName": "SharpFort.Template"
    }
  }
}
```

自动切换逻辑:
  sharpfort new MyMod
    -> 尝试 Primary (GitHub)
      成功 -> 使用 GitHub
      失败 -> 自动切换 Fallback
    -> 尝试 Fallback (Gitee)
      成功 -> 使用 Gitee（控制台提示）
      失败 -> 报错 + 建议运行 sharpfort doctor

doctor 诊断项:
  - 配置文件: 路径是否可读、JSON 格式是否正确
  - 缓存目录: 是否存在、占用空间、模板数量
  - 主源 GitHub: API 连通性、仓库存在性、分支可用性
  - 备用源 Gitee: API 连通性、仓库存在性
  - 认证状态: Token 是否配置、免认证频率限制提醒


## 实施计划

| 序号 | 功能 | 工作量 | 依赖 |
|:--:|------|:--:|------|
| 1 | 双源自动切换 (GitHub + Gitee) | 中 | 无 |
| 2 | init 命令 | 小 | 双源 |
| 3 | doctor 命令 | 中 | 双源 |
| 4 | list --refresh / --clear | 小 | 无 |

推荐顺序: 双源 -> init -> doctor -> list flags

## 实施后命令全景

| 命令 | 说明 |
|------|------|
| sharpfort init | 首次设置向导 |
| sharpfort new | 交互式创建模块 |
| sharpfort new <name> | 命令行创建 |
| sharpfort new <name> -s <分支> | 指定模板分支 |
| sharpfort new list | 列出模板 |
| sharpfort new list -d | 模板详细 |
| sharpfort new list -b <分支> | 预览结构 |
| sharpfort new list --refresh | 刷新缓存 |
| sharpfort new list --clear | 清空缓存 |
| sharpfort add-module <name> | 添加到 .sln |
| sharpfort clone | 克隆 SharpFort.Net |
| sharpfort clear | 清理 bin/obj |
| sharpfort doctor | 环境诊断 |
| sharpfort -v | 版本 |
| sharpfort -h | 帮助 |
| dotnet tool update -g sharpfort | 更新工具 |
