# Yi.Abp.Tool - 主CLI项目分析

## 📌 概述

**Yi.Abp.Tool** 是整个 CLI 工具的**入口项目**，也是最终打包为 .NET Global Tool 的项目。打包后的命令名为 `yi-abp`，版本 **2.0.5**，作者 **橙子老哥**，属于 yi-framework 框架的配套工具。

## 📁 文件结构与功能

### 1. Program.cs - 程序入口

**文件路径:** `Yi.Abp.Tool/Program.cs`

**核心职责:**
- 应用程序启动入口点
- 使用 `Host.CreateDefaultBuilder()` 创建通用主机
- 注册 ABP 模块 `YiAbpToolModule`
- 使用 Autofac 作为 DI 容器 (`UseAutofac()`)
- 通过 `CommandInvoker` 解析并执行命令行

**关键代码逻辑:**
```csharp
IHost host = Host.CreateDefaultBuilder()
    .ConfigureServices(async (host, service) => {
        await service.AddApplicationAsync<YiAbpToolModule>();
    })
    .UseAutofac()
    .Build();
var commandSelector = host.Services.GetRequiredService<CommandInvoker>();
await commandSelector.InvokerAsync(args);
```

**DEBUG 模式下的预设参数:**
```csharp
// args = ["-h"];            // 帮助
// args = ["-v"];            // 版本
// args = ["clear"];         // 清理
// args = ["new","oooo", "-t","module","-p","D:\temp","-csf"];  // 创建模块
// args = ["new","list"];    // 查看模板列表
// args = ["new","-h"];      // 查看子命令帮助
// args = ["add-module", "kkk"]; // 添加模块
```

### 2. ICommand.cs - 命令接口

**文件路径:** `Yi.Abp.Tool/ICommand.cs`

**核心职责:** 定义所有 CLI 命令必须实现的统一接口。

**接口定义:**
```csharp
public interface ICommand : ISingletonDependency
{
    string Command { get; }                          // 命令名称
    string? Description { get; }                     // 命令描述
    void CommandLineApplication(CommandLineApplication application);  // 配置命令选项
}
```

**设计说明:**
- 继承 `ISingletonDependency`，所有命令自动注册为**单例**
- `CommandLineApplication` 方法使用 `Microsoft.Extensions.CommandLineUtils` 的 API 来定义子命令、选项和参数

---

### 3. CommandInvoker.cs - 命令调用器

**文件路径:** `Yi.Abp.Tool/CommandInvoker.cs`

**核心职责:** 
- 扫描所有实现了 `ICommand` 接口的命令类
- 构建命令行应用程序树
- 分发并执行用户输入的命令

**关键逻辑:**

1. **构造函数注入** - 通过 DI 获取所有 `ICommand` 实现:
   ```csharp
   public CommandInvoker(IEnumerable<ICommand> commands)
   {
       _commands = commands;
       Application = new CommandLineApplication();
       InitCommand();
   }
   ```

2. **InitCommand()** - 初始化命令树:
   - 注册全局 `-h|--help` 选项
   - 注册全局 `-v|--versions` 版本选项
   - 遍历所有 `ICommand`，为每个命令创建子 CommandLineApplication
   
3. **InvokerAsync()** - 执行命令:
   ```csharp
   public async Task InvokerAsync(string[] args)
   {
       Application.Execute(args);
   }
   ```

## 📋 Commands 文件夹 - 命令实现

### 5. NewCommand.cs - 创建项目/模块命令

**命令名:** `new`  
**描述:** 创建项目模板 `yi-abp new <name> -csf`

**选项 (Options):**

| 选项 | 类型 | 说明 |
|------|------|------|
| `-t\|--template` | SingleValue | 模板类型: `module` 或 `porject`（默认: `module`） |
| `-p\|--path` | SingleValue | 创建路径 |
| `-csf` | NoValue | 是否创建解决方案文件夹 |
| `-s\|--soure` | SingleValue | 模板来源，gitee 模板库分支名称（默认: `default`） |
| `-dbms\|--dataBaseMs` | SingleValue | 数据库类型（MySQL, SQLite, SQLServer, Oracle, PostgreSQL） |

**子命令:**
- `new list` - 远程搜索所有可用模板列表

**参数:**
- `moduleName` - 模块名称（必须）

**执行流程:**
```
1. 检查 dbms 选项 → 提示在配置文件中修改连接串
2. 处理 path 选项 → 创建目录（若不存在）
3. 根据 templateType 调用模板生成服务
   - "module" → 调用 CreateModuleAsync
   - 其他 → 也调用 CreateModuleAsync（注：目前代码中两者逻辑相同）
4. 生成临时 zip 文件
5. 若 -csf → 创建解决方案文件夹（模块名转为小写+连字符）
6. 解压 zip 到目标目录
7. 删除临时 zip
```

**使用示例:**
```bash
# 创建模块（最简）
yi-abp new MyModule

# 创建模块到指定路径并创建解决方案文件夹
yi-abp new MyModule -t module -p D:\projects -csf

# 指定模板分支和数据库
yi-abp new MyModule -s dev-branch -dbms MySQL

# 查看可用模板
yi-abp new list
```

### 6. AddModuleCommand.cs - 添加模块命令

**命令名:** `add-module`  
**描述:** 将内容添加到当前解决方案 `yi-abp add-module <moduleName> [-p <path>] [-s <solution>]`

**选项:**

| 选项 | 类型 | 说明 |
|------|------|------|
| `-p\|--modulePath` | SingleValue | 模块路径（默认: 模块名小写+连字符） |
| `-s\|--solution` | SingleValue | 解决方案路径（默认: `../`） |

**参数:**
- `moduleName` - 模块名

**执行流程:**
```
1. 确定模块路径（默认 = 模块名.ToLower().Replace(".", "-")）
2. 确定 sln 路径（默认 = "../"）
3. CheckFirstSlnPath: 检查 sln 文件唯一性
4. CheckPathExist: 验证子项目目录存在
   → Application, Application.Contracts, Domain, Domain.Shared, SqlSugarCore
5. 执行 dotnet sln add 命令将子项目添加到解决方案
```

**涉及的子项目:**
- `{moduleName}.Application`
- `{moduleName}.Application.Contracts`
- `{moduleName}.Domain`
- `{moduleName}.Domain.Shared`
- `{moduleName}.SqlSugarCore`

**辅助方法:**
- `CheckFirstSlnPath()` - 校验 sln 文件（必须存在且唯一）
- `CheckPathExist()` - 校验各子项目目录存在
- `StartCmd()` - 跨平台执行 cmd/bash 命令

### 8. ClearCommand.cs - 清理命令

**命令名:** `clear`  
**描述:** 清除当前目录及子目录下的 obj、bin 文件夹 `yi-abp clear`

**选项:**

| 选项 | 类型 | 说明 |
|------|------|------|
| `-path` | SingleValue | 路径（默认: `./`） |

**黑名单目录:**
```csharp
List<string> delDirBlacklist = ["obj", "bin"];
```

**执行流程:**
1. 递归遍历指定目录及其所有子目录
2. 删除所有名为 `obj` 或 `bin` 的文件夹
3. 输出每个被删除的文件夹路径

**错误处理:**
- 删除失败时仅打印错误而不中断整体流程

## 🔧 命令行框架机制

```
yi-abp
├── -h|--help            全局帮助
├── -v|--versions        查看版本
├── new <name>           创建模板
│   ├── -t|--template   模板类型
│   ├── -p|--path       创建路径
│   ├── -csf            创建解决方案文件夹
│   ├── -s|--soure      模板分支来源
│   ├── -dbms|--dataBaseMs  数据库类型
│   └── list            列出所有模板
├── add-module <name>    添加模块
│   ├── -p|--modulePath 模块路径
│   └── -s|--solution   解决方案路径
├── clone                克隆源码
└── clear                清理 obj/bin
    └── -path           指定路径
```

## 🚀 C/S 架构说明

CLI 工具采用 **C/S（客户端-服务器）** 架构:

- **CLI 客户端** (`yi-abp`): 命令行入口，解析用户输入
- **Web API 服务端** (Yi.Abp.Tool.Web): 提供实际的模板生成等服务
- **通信方式:** HTTP 动态代理客户端 (`YiAbpToolHttpApiClientModule`)
- **生产环境:** `https://ccnetcore.com:19009`
- **开发环境:** `http://localhost:19002`
