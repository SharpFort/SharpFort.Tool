## 1. 双源自动切换 (GitHub + Gitee)

| `Domain/ConfigManager.cs` | `RepoConfig` 重构为 `Primary`/`Fallback` 双源结构；新增 `RepoSource`、`CloneConfig` 类；`CloneAddress` → `Clone.Primary`/`Clone.Fallback` |

## 2. init 命令

| 文件 | 说明 |
|------|------|
| `Commands/InitCommand.cs` | 4 步向导：克隆框架（双源）→ 模板仓库 Owner → Token 配置 → 确认保存；支持 `--skip-clone` |

交互流程:
1. 提示 git clone SharpFort.Net（主源失败自动切换备用源）
2. 说明框架提供的基础能力 (RBAC/审计日志/限流/多租户/定时任务)
3. 模板仓库 Owner 确认（默认 SharpFort）
4. GitHub Token（可选）
5. 自动生成 ~/.sharpfort/config.json

---

## 3. doctor 命令

| 文件 | 说明 |
|------|------|
| `Commands/DoctorCommand.cs` | 6 项诊断：配置文件 → 缓存目录 → 主源连通 → 备用源连通 → Token 状态 → Git 可用性；汇总通过/警告/失败计数 |

诊断项:
- 配置文件: 路径是否可读、JSON 格式是否正确
- 缓存目录: 是否存在、占用空间、模板数量
- 主源 GitHub: API 连通性、仓库存在性、分支可用性
- 备用源 Gitee: API 连通性、仓库存在性
- 认证状态: Token 是否配置、免认证频率限制提醒
- 工具依赖: git 是否可用
