# Yi.Abp.Tool.Domain - 领域层分析

## 📌 概述

**Yi.Abp.Tool.Domain** 是 ABP 分层架构中的**领域层**（Domain Layer），包含核心业务逻辑。这是整个工具最关键的层，实现了模板生成、Gitee API 交互、NuGet 版本爬虫等核心功能。

**项目路径:** `tool/Yi.Abp.Tool.Domain/`

|------|-----|
| **TargetFramework** | net8.0 |
| **输出类型** | Class Library（类库） |

## 📁 文件结构与功能

### 1. YiAbpToolDomainModule.cs - 模块定义

```csharp
[DependsOn(typeof(YiAbpToolDomainSharedModule))]
public class YiAbpToolDomainModule : AbpModule
{
    public override void ConfigureServices(ServiceConfigurationContext context)
    {
        var configuration = context.Services.GetConfiguration();
        Configure<ToolOptions>(configuration.GetSection("ToolOptions"));
        var toolOptions = new ToolOptions();
        configuration.GetSection("ToolOptions").Bind(toolOptions);
        if (!Directory.Exists(toolOptions.TempDirPath))
        {
            Directory.CreateDirectory(toolOptions.TempDirPath);
        }
    }
}
```

**启动时行为:**
1. 从配置文件 `appsettings.json` 的 `ToolOptions` 节读取配置
2. 自动创建 `TempDirPath` 临时目录（若不存在）

#### 📌 方法一: CreateTemplateAsync() - 创建模板（核心算法）

```csharp
public async Task<string> CreateTemplateAsync(TemplateGenCreateDto input)
```

**完整执行流程:**

```
┌─────────────────────────────────────────────────────┐
│  Step 1: 检查 Gitee 分支是否存在                      │
│    → GiteeManager.IsExsitBranchAsync(input.GiteeRef) │
│    → 不存在则抛出 UserFriendlyException              │
├─────────────────────────────────────────────────────┤
│  Step 2: 检查临时目录配置                              │
│    → ToolOptions.TempDirPath 是否为空                │
├─────────────────────────────────────────────────────┤
│  Step 3: 创建临时工作目录                              │
│    → {TempDirPath}/{GUID}/                           │
├─────────────────────────────────────────────────────┤
│  Step 4: 创建下载目录                                  │
│    → {TempDirPath}/download/                         │
├─────────────────────────────────────────────────────┤
│  Step 5: 从 Gitee 下载模板 zip                        │
│    → GiteeManager.DownLoadFileAsync(input.GiteeRef)  │
│    → 保存为 {downloadPath}/{GUID}.zip                 │
├─────────────────────────────────────────────────────┤
│  Step 6: 解压 zip 到临时目录                           │
│    → ZipFile.ExtractToDirectory()                   │
├─────────────────────────────────────────────────────┤
│  Step 7: 获取解压后的操作目录（zip内多一层）              │
│    → operPath = Directory.GetDirectories(tempFile)[0]│
├─────────────────────────────────────────────────────┤
│  Step 8: 替换模板内容 ⭐                              │
│    → ReplaceContentAsync(operPath, ReplaceStrData)  │
│    → 递归替换: 目录名 + 文件名 + 文件内容               │
├─────────────────────────────────────────────────────┤
│  Step 9: 重新打包为 zip                                │
│    → ZipFile.CreateFromDirectory()                  │
├─────────────────────────────────────────────────────┤
│  Step 10: 清理临时目录                                 │
│    → Directory.Delete(tempFileDirPath)               │
├─────────────────────────────────────────────────────┤
│  返回: 最终 zip 文件路径                               │
└─────────────────────────────────────────────────────┘
```

#### 📌 ReplaceContentAsync() - 模板内容替换算法 ⭐

```csharp
private async Task ReplaceContentAsync(string rootDirectory, Dictionary<string, string> dic)
```

**递归替换逻辑:**

```
ForEach (Key, Value) in ReplaceStrData:
    ReplaceInDirectory(rootDir, Key, Value)
        ├── ReplaceInFiles(dirPath, Key, Value)
        │   ├── 替换目录名（Directory.Move）
        │   ├── 替换文件名（File.Move）
        │   └── 替换文件内容（ReadAllText → Replace → WriteAllText）
        └── 递归处理子目录
```

**替换范围:**
- ✅ 目录名称
- ✅ 文件名称
- ✅ 文件内容（文本）

**替换键值对示例:**
```
"Yi.Abp"  → "MyCompany.MyModule"
"YiAbp"   → "MyCompanyMyModule"
```

#### 📌 GetAllTemplatesAsync() - 获取模板列表

```csharp
public async Task<List<string>> GetAllTemplatesAsync()
```

**执行流程:**
1. 调用 `GiteeManager.GetAllBranchAsync()` 获取所有分支
2. 移除 `master` 分支
3. 返回其他分支名作为模板列表

> 每个 Gitee 分支代表一种模板类型。

### 3. GiteeManager.cs - Gitee API 管理器 ⭐

**文件路径:** `Yi.Abp.Tool.Domain/GiteeManager.cs`

**注册方式:** `ITransientDependency`

**常量配置:**
```csharp
private const string GiteeHost = "https://gitee.com/api/v5";
private const string Owner = "ccnetcore";
private const string Repo = "yi-template";
```

**模板仓库:** `https://gitee.com/ccnetcore/yi-template`

**认证配置:**
```csharp
_accessToken = configuration.GetValue<string>("GiteeAccession");
```
> 从 `appsettings.json` 的 `GiteeAccession` 读取访问令牌。

#### 📌 方法一: IsExsitBranchAsync() - 检查分支是否存在

```csharp
public async Task<bool> IsExsitBranchAsync(string branch)
```

**API 调用:**
```
GET https://gitee.com/api/v5/repos/ccnetcore/yi-template/branches/{branch}?access_token={token}
```

**返回:**
- `true` - 分支存在
- `false` - 404 Not Found

#### 📌 方法二: GetAllBranchAsync() - 获取所有分支

```csharp
public async Task<List<string>> GetAllBranchAsync()
```

**API 调用:**
```
GET https://gitee.com/api/v5/repos/ccnetcore/yi-template/branches?access_token={token}&sort=name&direction=asc&page=1&per_page=100
```

**处理流程:**
1. 调用 Gitee API v5 接口
2. 解析返回的 JSON 数组
3. 提取每个分支的 `name` 字段
4. 返回分支名列表

#### 📌 方法三: DownLoadFileAsync() - 下载分支代码

```csharp
public async Task<Stream> DownLoadFileAsync(string branch)
```

**API 调用:**
```
GET https://gitee.com/api/v5/repos/ccnetcore/yi-template/zipball?access_token={token}&ref={branch}
```

**返回:** 文件流（Stream），为整个仓库分支的 zip 包

### 4. NugetCrawlerManager.cs - NuGet 爬虫管理器

**文件路径:** `Yi.Abp.Tool.Domain/NugetCrawlerManager.cs`

**注册方式:** `ITransientDependency`

**目标 URL:**
```csharp
private const string NugetVersionUrl = "https://www.nuget.org/packages/Yi.Abp.Tool#versions-body-tab";
```

#### 🕷️ 爬虫机制

**初始化（构造函数）:**
```csharp
public NugetCrawlerManager(IDistributedCache<NugetResult> cache)
{
    this.NugetResult = cache.GetOrAdd("NugetResult", 
        () => InitData(),    // 缓存未命中时的工厂方法
        () => {              // 缓存选项
            options.AbsoluteExpiration = DateTimeOffset.Now.AddMinutes(1);
        });
}
```

**缓存策略:**
- 使用 ABP 分布式缓存（`IDistributedCache`）
- 缓存键: `"NugetResult"`
- 过期时间: **1 分钟**

#### 📌 InitData() - 初始化爬虫数据

```
1. 加载 NuGet 页面 HTML
   HtmlWeb.Load("https://www.nuget.org/packages/Yi.Abp.Tool#versions-body-tab")

2. GetVersionList() - 解析版本列表
   XPath: //*[@id="version-history"]/table/tbody
   → 遍历 tr → td → a → title 属性

3. GetDownloadNumber() - 解析下载总数
   XPath: //*[@id="skippedToContent"]/section/div/aside/div[1]/div[2]/div[1]/span[2]
   → 处理 "K" 后缀（如 "1.5K" → 1500）
```

#### 📌 NugetResult 数据结构

```csharp
public class NugetResult
{
    public long DownloadNumber { get; set; }      // 下载总数
    public List<string> Versions { get; set; }    // 版本历史列表
}
```
