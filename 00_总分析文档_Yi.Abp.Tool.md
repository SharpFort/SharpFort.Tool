## 总体架构

### 架构模式: ABP DDD 分层 + C/S 架构

项目采用 ABP 框架标准的 DDD 分层架构，一共 7 个子项目。

| 层级 | 项目 | 职责 |
|------|------|------|
| 表示层 | Yi.Abp.Tool | CLI 入口、命令解析、用户交互 |
| Web 服务层 | Yi.Abp.Tool.Web | RESTful API 服务、Swagger 文档 |
| HTTP 代理层 | Yi.Abp.Tool.HttpApi.Client | 动态生成 C# HTTP 客户端代理 |
| 应用服务层 | Yi.Abp.Tool.Application | 用例协调、DTO 映射 |
| 应用契约层 | Yi.Abp.Tool.Application.Contracts | 服务接口、输入 DTO |
| 领域层 | Yi.Abp.Tool.Domain | 核心业务逻辑、Gitee API、爬虫 |
| 共享领域层 | Yi.Abp.Tool.Domain.Shared | 枚举、共享 DTO、配置选项 |

### C/S 通信架构

CLI 客户端通过 ABP 动态 HTTP 代理调用远程 Web API 服务器。
- 生产环境: https://ccnetcore.com:19009
- 开发环境: http://localhost:19002

## 命令清单

| 命令 | 描述 | 使用示例 |
|------|------|----------|
| `new` | 创建项目/模块模板 | `yi-abp new MyModule -csf` |
| `new list` | 查看所有可用模板 | `yi-abp new list` |
| `add-module` | 添加模块到解决方案 | `yi-abp add-module MyModule -s ../` |
| `clone` | 克隆 Yi 框架源码 | `yi-abp clone` |
| `clear` | 清理 bin/obj 目录 | `yi-abp clear` |

| 命令 | 选项 | 类型 | 默认值 | 说明 |

| `new` | `-t\|--template` | string | `module` | 模板类型 |
| `new` | `-p\|--path` | string | `./` | 创建路径 |

| `add-module` | `-s\|--solution` | string | `../` | 解决方案路径 |

## 配置项总览

### appsettings.json 完整配置

```json
{
  "App": {
    "SelfUrl": "http://*:19002",
    "CorsOrigins": "http://localhost:18001;https://ccnetcore.com"
  },
  "ToolOptions": {
    "TempDirPath": "wwwroot/temp",
    "ModuleTemplateFilePath": "wwwroot/ModuleTemplate.zip",
    "ProjectTemplateFilePath": "wwwroot/ProjectTemplate.zip"
  },
  "GiteeAccession": ""
}
```

### 配置项说明

| 配置项 | 层级 | 必需 | 说明 |
|--------|------|:--:|------|
| `App:SelfUrl` | Web | Y | 服务监听地址 |
| `App:CorsOrigins` | Web | Y | 跨域白名单(分号分隔) |
| `ToolOptions:TempDirPath` | Domain | Y | 模板临时文件目录 |
| `ToolOptions:ModuleTemplateFilePath` | Domain | N | 模块模板路径(预留) |
| `ToolOptions:ProjectTemplateFilePath` | Domain | N | 项目模板路径(预留) |
| `GiteeAccession` | Domain | Y | Gitee API 访问令牌 |
| `AbpRemoteServiceOptions` | CLI | Y | 远程 API 地址(硬编码) |

## 核心业务流程

### 1. 模板生成流程 (new 命令)

```
用户执行: yi-abp new MyModule -t module -csf
  |
  +-- Step 1: NewCommand 解析命令行参数
  +-- Step 2: 构建 TemplateGenCreateInputDto, SetNameReplace()
  |             Yi.Abp -> MyModule, YiAbp -> MyModule
  +-- Step 3: HTTP 调用远程 API (POST /api/app/template-gen)
  +-- Step 4: [服务端] GiteeManager.IsExsitBranchAsync()
  +-- Step 5: [服务端] GiteeManager.DownLoadFileAsync()
  +-- Step 6: [服务端] 解压模板, ReplaceContentAsync()
  |             递归替换: 目录名 + 文件名 + 文件内容
  +-- Step 7: [服务端] 重新打包返回 byte[]
  +-- Step 8: [客户端] 保存 zip, 解压到目标目录
  |            若 -csf -> 创建解决方案文件夹
  +-- Step 9: 完成! 输出: 恭喜~模块已生成!
```

### 2. 添加模块流程 (add-module 命令)

```
用户执行: yi-abp add-module MyModule -s ../
  |
  +-- Step 1: 确定模块路径 (MyModule -> mymodule)
  +-- Step 2: 查找 .sln 文件, 校验唯一性
  +-- Step 3: 校验子项目目录存在
  |             mymodule/MyModule.Application
  |             mymodule/MyModule.Application.Contracts
  |             mymodule/MyModule.Domain
  |             mymodule/MyModule.Domain.Shared
  |             mymodule/MyModule.SqlSugarCore
  +-- Step 4: 执行 dotnet sln add 命令
```

### 3. 克隆流程 (clone 命令)

```
yi-abp clone -> git clone https://gitee.com/ccnetcore/Yi
```

### 4. 清理流程 (clear 命令)

```
yi-abp clear -> 递归删除所有 bin/ 和 obj/ 目录
```

## 扩展指南

### 添加新命令

1. 在 `Yi.Abp.Tool/Commands/` 下创建新类，实现 `ICommand` 接口
2. 通过 DI 注入所需服务（如 `ITemplateGenService`）
3. 命令自动被 `CommandInvoker` 发现和注册

```csharp
public class MyNewCommand : ICommand
{
    public string Command => "my-command";
    public string? Description => "我的新命令描述";
    public void CommandLineApplication(CommandLineApplication app)
    {
        app.OnExecute(() => { /* 逻辑 */ return 0; });
    }
}
```

### 添加新配置项

1. 在 `ToolOptions.cs` 中添加新属性
2. 在 `appsettings.json` 的 `ToolOptions` 节添加配置
3. 确认 `YiAbpToolDomainModule.ConfigureServices` 中已完成绑定

### 添加新 API

1. 在 `ITemplateGenService` 或新建接口中定义方法
2. 在 Application 层实现
3. ABP 动态 API 自动暴露为 HTTP 端点

### 本地开发调试

1. 启动 Web 项目: `cd Yi.Abp.Tool.Web && dotnet run`
2. 修改 CLI 远程地址: 将 `YiAbpToolModule.cs` 中地址改为 `http://localhost:19002`
3. 使用 `yi-abp` 命令测试
