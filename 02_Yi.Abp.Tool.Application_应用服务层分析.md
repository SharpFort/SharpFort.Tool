# Yi.Abp.Tool.Application - 应用服务层分析

## 📌 概述

**Yi.Abp.Tool.Application** 是 ABP 分层架构中的**应用服务层**（Application Layer），负责协调领域对象来完成用例，是 CLI 命令与领域逻辑之间的桥接层。该层实现了 DDD 中的应用服务模式。

**项目路径:** `tool/Yi.Abp.Tool.Application/`

## 🏗️ 项目信息

| 属性 | 值 |
|------|-----|
| **TargetFramework** | net8.0 |
| **输出类型** | Class Library（类库） |

### 2. TemplateGenService.cs - 模板生成服务 ⭐核心

**文件路径:** `Yi.Abp.Tool.Application/TemplateGenService.cs`

**实现接口:** `ITemplateGenService`

**依赖注入:**
```csharp
private readonly TemplateGenManager _templateGenManager;
```

#### 📌 方法一: CreateModuleAsync() - 创建模块

```csharp
public async Task<byte[]> CreateModuleAsync(TemplateGenCreateInputDto moduleCreateInputDto)
```

**执行流程:**
```
1. moduleCreateInputDto.SetNameReplace()
   → 构建 ReplaceStrData 字典
   → "Yi.Abp" → Name
   → "YiAbp" → Name.Replace(".", "")

2. input.SetTemplateGiteeRef(moduleCreateInputDto.ModuleSoure)
   → 将模块分支来源转为小写作为 Gitee 分支引用

3. 调用 TemplateGenManager.CreateTemplateAsync(input)
   → 返回生成的临时 zip 文件路径

4. 读取 zip 文件内容返回 byte[]
```

**DTO 转换流程:**
```
TemplateGenCreateInputDto (Contracts 层 DTO)
    ↓ Adapt<>() (Mapster 自动映射)
TemplateGenCreateDto (Domain.Shared 层 DTO)
    ↓ SetTemplateGiteeRef()
最终领域 DTO → Manager 层使用
```

#### 📌 方法二: GetAllTemplatesAsync() - 获取全部模板列表

```csharp
[HttpGet("template-gen/template")]
public async Task<List<string>> GetAllTemplatesAsync()
```

**执行流程:**
1. 调用 `TemplateGenManager.GetAllTemplatesAsync()`
2. 返回所有可用模板名称列表

**特性:**
- 标记了 `[HttpGet("template-gen/template")]`，可被 ABP 动态 API 自动暴露为 HTTP 端点

### 3. NueGetInfoService.cs - NuGet 信息服务

**文件路径:** `Yi.Abp.Tool.Application/NueGetInfoService.cs`

**依赖注入:**

**方法:**

#### GetInfo() - 获取 NuGet 爬虫结果

```csharp
public NugetResult GetInfo()
{
    return _nugetCrawlerManager.GetNugetResult();
}
```

**返回类型 `NugetResult`:**
```csharp
public class NugetResult

}
```

> ⚠️ **注意:** 此服务目前未在 CLI 命令中暴露使用，属于 Web API 暴露的服务。

## 🔗 层间关系

```
┌─────────────────────────────────────────┐
│          CLI Commands (Yi.Abp.Tool)      │  ← 用户交互层
├─────────────────────────────────────────┤
│  Application.Contracts (接口 + DTO)      │  ← 契约层
│    - ITemplateGenService                │
│    - TemplateGenCreateInputDto          │
├─────────────────────────────────────────┤
│  Application (本层)                      │  ← 应用服务实现
│    - TemplateGenService                 │
│    - NueGetInfoService                  │
├─────────────────────────────────────────┤
│  Domain                                 │  ← 领域逻辑
│    - TemplateGenManager                 │
│    - GiteeManager                       │
│    - NugetCrawlerManager                │
└─────────────────────────────────────────┘
```

## 📝 设计模式与最佳实践

1. **应用服务模式** - 继承 `ApplicationService`，获得 ABP 基础设施支持
2. **DTO 分离** - Input DTO 在 Contracts 层定义，领域 DTO 在 Domain.Shared 层
3. **Mapster 映射** - 使用 `Adapt<>()` 进行对象映射，减少手动赋值
4. **依赖倒置** - 依赖接口 `ITemplateGenService` 和 `TemplateGenManager`
